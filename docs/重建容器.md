# 重建容器流程（PuTTY / SSH 一次完成）

這份文件提供「可直接貼到 PuTTY」的完整流程，用來重建並啟動 `opendata-ua` 服務。

## 前提

1. 你已能用 SSH 登入 NAS。
2. 專案目錄內有以下檔案：

- `docker-compose.yml`
- `Dockerfile`
- `main.py`
- `server/`
- `webui/`
- `data/config.json`
- `setup.sh`

## 一次貼上執行（建議）

以下路徑已直接使用實際專案路徑，可整段貼到 PuTTY：

```bash
cd /volume1/web_packages/docker/opendataUA-nas

# 0) 前置檢查（缺檔就中止）
test -f docker-compose.yml || { echo "缺少 docker-compose.yml"; exit 1; }
test -f Dockerfile || { echo "缺少 Dockerfile"; exit 1; }
test -f main.py || { echo "缺少 main.py"; exit 1; }
test -d server || { echo "缺少 server/"; exit 1; }
test -d webui || { echo "缺少 webui/"; exit 1; }
test -f data/config.json || { echo "缺少 data/config.json"; exit 1; }

# 1) 修正 shell script 權限與 Windows 換行
sed -i 's/\r$//' setup.sh update.sh
chmod +x setup.sh update.sh

# 2) 執行重建與啟動
./setup.sh

# 3) 查看狀態
docker compose ps

# 4) 看啟動日誌（按 Ctrl+C 離開）
docker compose logs -f --tail=200 opendata-ua
```

## 每個步驟在做什麼

1. `sed -i 's/\r$//' setup.sh update.sh`

- 移除 Windows 換行，避免 NAS 執行 `.sh` 時出現 `bad interpreter`。

2. `chmod +x setup.sh update.sh`

- 讓 NAS 可以直接執行 shell script。

3. `./setup.sh`

- 使用唯一設定檔 `data/config.json`。
- 建立 `.env`（如果不存在）。
- 停掉舊容器。
- 重新 build Docker image。
- 啟動 `opendata-ua` container。
- 檢查 `http://127.0.0.1:8188/api/health`。

4. `docker compose logs -f --tail=200 opendata-ua`

- 立刻驗證容器啟動是否正常（有錯可馬上看到 traceback）。

## 驗證容器是否正常

```bash
cd /volume1/web_packages/docker/opendataUA-nas
docker compose ps
curl -I http://127.0.0.1:8188/api/health
```

瀏覽器開啟：

```text
http://<NAS-IP>:8188
```

OPC UA endpoint：

```text
opc.tcp://<NAS-IP>:48484
```

## 常用排錯命令

```bash
cd /volume1/web_packages/docker/opendataUA-nas

# 看目前容器狀態
docker compose ps

# 看最近 200 行日誌
docker compose logs --tail=200 opendata-ua

# 持續追蹤日誌
docker compose logs -f opendata-ua

# 進入容器
docker compose exec opendata-ua sh

# 在容器內看 Python 套件清單
docker compose exec opendata-ua python -m pip list

# 僅重啟容器
docker compose restart opendata-ua
```

## 重要提醒

1. 目前 compose 有掛載 `main.py`、`server/`、`webui/`，代表程式碼會對映進容器。
2. Python 套件不是掛載資料，仍要靠 build 重新安裝。
3. 改到 `Dockerfile` 或 Python 套件時，請重建容器。
4. Web UI 預設 port 是 `8188`。
