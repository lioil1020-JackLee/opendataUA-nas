# 更新容器流程（PuTTY / SSH 一次完成）

這份文件提供「可直接貼到 PuTTY」的完整流程，用來套用已上傳到 NAS 的程式碼更新。

## 前提

1. 你已能用 SSH 登入 NAS。
2. 專案已經跑過 `setup.sh` 或已存在可用的 `docker-compose.yml`。
3. 專案目錄內有以下檔案：
- `docker-compose.yml`
- `Dockerfile`
- `main.py`
- `server/`
- `webui/`
- `update.sh`

## 一次貼上執行（建議）

以下路徑已直接使用實際專案路徑，可整段貼到 PuTTY：

```bash
cd /volume1/web_packages/docker/opendataUA-nas

# 0) 前置檢查（缺檔就中止）
test -f docker-compose.yml || { echo "缺少 docker-compose.yml"; exit 1; }
test -f Dockerfile || { echo "缺少 Dockerfile"; exit 1; }
test -f main.py || { echo "缺少 main.py"; exit 1; }
test -d server || { echo "缺少 server/"; exit 1; }
test -d webui || { echo "缺少 webui/"; exit 1; }
test -f update.sh || { echo "缺少 update.sh"; exit 1; }

# 1) 修正 shell script 權限與 Windows 換行
sed -i 's/\r$//' update.sh
chmod +x update.sh

# 2) 套用一般程式碼更新
./update.sh

# 3) 查看狀態
docker compose ps

# 4) 看啟動日誌（按 Ctrl+C 離開）
docker compose logs -f --tail=200 opendata-ua
```

## 每個步驟在做什麼

1. `sed -i 's/\r$//' update.sh`
- 移除 Windows 換行，避免 NAS 執行 `.sh` 時出現 `bad interpreter`。

2. `chmod +x update.sh`
- 讓 NAS 可以直接執行 update script。

3. `./update.sh`
- 確認 `data/config.json` 存在。
- 使用目前 image 重新建立 container。
- 因為 compose 有掛載 `main.py`、`server/`、`webui/`，一般程式碼更新不一定要重建 image。
- 檢查 `http://127.0.0.1:8188/api/health`。

4. `docker compose logs -f --tail=200 opendata-ua`
- 立刻驗證容器啟動是否正常。

## 如果有改 Dockerfile 或 Python 套件

如果你修改了 `Dockerfile`，或新增/移除 Python 套件，請改用：

```bash
cd /volume1/web_packages/docker/opendataUA-nas
chmod +x update.sh
./update.sh --build
```

## 驗證服務

```bash
cd /volume1/web_packages/docker/opendataUA-nas
docker compose ps
curl -I http://127.0.0.1:8188/api/health
```

瀏覽器開啟：

```text
http://<NAS-IP>:8188
```

OPC UA endpoint：

```text
opc.tcp://<NAS-IP>:48484
```

## 常用排錯命令

```bash
cd /volume1/web_packages/docker/opendataUA-nas

# 看目前容器狀態
docker compose ps

# 看最近 200 行日誌
docker compose logs --tail=200 opendata-ua

# 持續追蹤日誌
docker compose logs -f opendata-ua

# 僅重啟容器
docker compose restart opendata-ua

# 重新 build 並啟動
./update.sh --build
```

## 重要提醒

1. 一般更新程式碼，用 `./update.sh`。
2. 改到 `Dockerfile` 或 Python 套件，用 `./update.sh --build`。
3. `data/config.json` 是 NAS 上的持久化設定，不建議每次更新都覆蓋。
4. Web UI 預設 port 是 `8188`。
